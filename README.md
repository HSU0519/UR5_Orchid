# UR5 Orchid Picking System

## 專案簡介
這是一個基於 UR5 機器手臂、Robotiq 2F-85 夾爪與 RealSense D435i 深度相機開發的「蘭花自動夾取與分類系統」。本系統結合了 YOLO 物件偵測、姿態估計 (Pose Estimation) 與 PnP Eye-to-Hand 校正技術，能夠自動辨識一般蘭花及帶病徵的蘭花，計算適當的夾取角度與位置，並執行精確的夾取與放置任務。

## 主要功能
* **一般蘭花姿態對齊夾取**：使用 YOLO 偵測蘭花姿態，機器手臂能自動旋轉夾爪對齊蘭花進行夾取。
* **病徵蘭花自動連續夾取**：自動偵測帶有病徵的蘭花，計算根部偏移量，並將其夾取至預先設定好的苗架或收集區。
* **動態網格放置 (XYZ 空間補償)**：透過記錄 4 個基準點，系統能自動進行雙線性插值 (Bilinear Interpolation)，生成 3x5 (共 15 個) 的放置網格孔位，並支援動態 Z 軸高度補償。
* **即時 Eye-to-Hand 4x4 PnP 校正**：無須複雜的多高度調整，透過單張棋盤格影像與 4 個探針點，即可快速計算並更新 4x4 空間座標轉換矩陣 (`C44_eyetohand.npy`)，解決空間視差與定位誤差問題。
* **Ant Colony Optimization (ACO)**：內建蟻群演算法優化連續夾取多株蘭花的最短路徑。

## 硬體需求
* UR5 機器手臂 (控制器 IP: 預設 `192.168.1.101`)
* Robotiq 2F-85 智慧夾爪
* Intel RealSense D435i 深度相機
* 具有獨立顯示卡 (NVIDIA GPU) 之運算主機 (供 YOLO 推論使用)

## 專案結構
```text
UR5-orchid/
├── models/                     # 存放 YOLO 與 Pose Estimation 權重檔 (.pt, .onnx)
├── src/                        # 主要程式碼目錄
│   ├── YOLO_detect+rotate.py   # 系統主程式 (操作控制、影像辨識、手臂運動規劃)
│   ├── pnp_calibration.py      # Eye-to-Hand PnP 校正程式
│   ├── orchid_pose_d435.py     # 結合 D435i 的蘭花姿態與分割推論核心
│   ├── function_arm.py         # 機器手臂旋轉/座標轉換輔助函式
│   ├── algorithm/              # 演算法資料夾 (如 ACO 蟻群演算法路徑最佳化)
│   ├── calibration_data/       # 存放產生的 4x4 校正矩陣 (C44_eyetohand.npy)
│   └── data/                   # 存放動態紀錄檔 (如放置計數器、XYZ 點位資料)
└── data/                       # 存放拍攝的影像 (A) 與深度數據 (D)、姿態紀錄
```

## 系統操作說明

啟動主程式前，請確保位於專案根目錄，並已啟用包含相依套件的虛擬環境：
```bash
python src/YOLO_detect+rotate.py
```

在啟動後的 RealSense 影像視窗中，可使用以下鍵盤快捷鍵進行控制：

### 核心操作
* `Esc` : 結束程式並釋放資源
* `q` / `Q` : 機器手臂回歸初始預備姿態 (Home Pose)
* `x` / `X` : 拍照並紀錄當前機器手臂座標與影像/深度資料
* `r` / `R` : 偵測畫面上的一般蘭花，並執行一次性夾取路徑
* `p` / `P` : 啟動**自動連續辨識並夾取**所有病徵蘭花的模式
* `b` / `B` : 強制取消自動夾取模式
* `c` / `C` : 執行 PnP Eye-to-Hand 校正並即時更新轉換矩陣

### 放置點紀錄 (3x5 網格)
* `z` / `Z` : 紀錄當前夾爪的空間座標 (XYZ) 作為放苗孔端點。
  > **注意**：系統需連續記錄 4 個端點才能生成 15 個洞位。請手動依序引導手臂至苗架的 **1號孔 -> 3號孔 -> 13號孔 -> 15號孔**，並在各點按下 `z` 記錄。
* `0` (數字零) : 將連續放置的計數器歸零。

### 手動微調 (Jogging)
* `w` / `s` / `a` / `d` / `8` / `2` : 手動控制手臂在 X / Y / Z 軸向移動。
* `j` / `k` / `l` : 切換手動移動的位移量 (Step Size) 分別為 `10mm` / `1mm` / `0.1mm`。

## 校正流程 (Calibration)
1. 將棋盤格放置於工作平面，並手動引導手臂探針接觸棋盤格的 4 個特定角落，透過系統記錄其 3D 座標產生 `pose.txt`。
2. 在程式執行中按下 `x` 拍攝一張清晰的棋盤格影像。
3. 按下 `c` 觸發 `src/pnp_calibration.py` 腳本。系統將：
   - 讀取 4 個探針點，用 3D 向量展開 132 個基準世界座標。
   - 使用 OpenCV 偵測影像中棋盤格的像素座標。
   - 透過 `cv2.solvePnP` 解算出精確的 4x4 轉換矩陣。
   - 將新矩陣存入 `src/calibration_data/C44_eyetohand.npy` 並自動於主程式內動態載入生效。

## 注意事項
* 確保 UR5 手臂連線正常 (預設 IP: `192.168.1.101`)。
* 執行前如遭遇缺少模組，請確保 `urx`, `pyrealsense2`, `opencv-python`, `numpy` 等套件已安裝。
* 若發生夾取深度誤差，可於 `src/YOLO_detect+rotate.py` 內調整 `PICK_Z` 或 `PUT_Z_OFFSET` 等全局高度參數。
